What is Kurdistan?

A generated-transport compiler with an adaptive-runtime direction.

Compiler layer

Profile-specific transport behavior

Profiles define generated source backends, frame grammars, schedulers, padding behavior, stream semantics, probing behavior, malformed-input policy, and audit-visible safe metadata.

Adaptive runtime direction

Candidate paths under volatile conditions

The adaptive model represents path families, freshness, uncertainty, generated bundles, future racing, scoring, monitoring, and failover without implying a finished proxy or VPN product.

Why this exists

Censorship can be both fingerprint-based and fast-changing.

In heavily filtered environments, a path may work on one device and fail on another, or work briefly and fail seconds later. Kurdistan treats censorship not only as a fingerprinting problem, but also as a fast-changing path-selection problem.

Iran is one motivating example of a volatile censorship environment, but the project does not hard-code a country-specific mode. The work remains a research prototype using synthetic, payload-free evaluation.

Adaptive runtime direction

The target is not one fixed best transport.

The long-term goal is to generate candidate bundles, evaluate short-lived path evidence, race candidates, monitor health, and fail over when conditions change.

Boundaries and safety

Safety controls stay visible in the public face of the project.

Payload-free audit traces
No raw packet capture fixtures
No secrets, keys, nonces, auth tags, or proof material in traces
No bypass guarantees
No undetectability claims
Lab readiness is distinct from readiness for field use
High-risk carriers require separate review
No concrete proxy, VPN, TLS, CDN, or public-network carrier implementation yet
