# KIP-0010: Adversarial Lab Simulator

Structural profile diversity is necessary but not enough. Two profiles can differ in JSON and still produce traces that look like one family from the outside. The adversarial lab simulator treats trace events as black-box observations and asks whether generated traces collapse into tight observable clusters.

This is a local-only heuristic. It does not prove undetectability, production safety, or real-world censorship resistance.

## Black-Box Features

The simulator extracts payload-free features from trace events:

- first frame size and first N frame size buckets
- first-contact event count
- direction sequence and direction-change count
- canonical state path shape and path length, without raw state names
- semantic event count, where trace metadata has it
- total frame count, total bytes, payload byte buckets, and padding byte buckets
- inter-event timing buckets
- close, invalid-input, and malformed-frame outcome buckets
- scheduler mode pattern
- frame-size histogram
- burst count and approximate upload/download ratio
- session duration bucket

Feature vectors do not include payload contents, secrets, keys, proofs, raw frames, wire symbols, external target data, or raw generated state names.

## Distance And Clustering

Distance is deterministic and intentionally simple. Numeric features use normalized absolute difference. Categorical buckets use exact-match comparison. Core observable shape features have higher weight than timing and padding buckets so random padding alone does not dominate the result.

Clustering uses a thresholded distance graph and connected components. The implementation is deliberately small and dependency-free.

## Synthetic Controls

The simulator includes calibration controls:

- `fixed_protocol`: identical first-contact structure, fixed frame sizes, fixed close behavior, and fixed invalid-input behavior. It should cluster tightly and be flagged as fixed-signature.
- `noisy_fixed_protocol`: fixed core sequence with varied padding and timing. It should still cluster as a fixed family, showing that random padding alone is not enough.
- `random_byte_protocol`: random sizes and timing without coherent state structure. It is a noisy baseline, not proof of useful polymorphism.
- `raw_echo_baseline`: simple shape baseline for local echo-like traces.

## Audit Gate

`kcheck` runs `adversarial_black_box_clustering` as a required gate. It fails when:

- generated traces collapse into one tight cluster
- different profiles are too close under black-box distance
- same-profile traces do not cluster together
- fixed controls are not detected as fixed
- noisy-fixed controls are not detected as suspiciously fixed-family

## Commands

Run the standalone adversary analysis:

```bash
go run ./cmd/kcheck adversary --quick
```

Write JSON:

```bash
go run ./cmd/kcheck adversary --quick --out testdata/audit/adversary.json
```

Run the normal audit, which includes the adversarial gate:

```bash
go run ./cmd/kcheck --quick
go run ./cmd/kcheck --full --out testdata/audit/latest.json
```

## Limitations

The simulator is not a real censor, not a classifier benchmark, and not a proof of resistance to traffic analysis. It sees only local trace metadata generated by this lab harness. Its thresholds are regression gates, not scientific guarantees. A pass means the current corpus did not trigger known local collapse checks. It does not mean the protocol family is safe to deploy or difficult to detect in real networks.

The project remains loopback-only and lab-only. It does not add VPN mode, SOCKS, HTTP carriers, TLS mimicry, external targets, deployment scripts, mobile clients, live-network testing, or operational evasion guidance.
